Question-01:Discuss the scope of var, let, and const.

The scope of variables in JavaScript is determined by their declaration keywords:

var Scope:

The var keyword introduces variables with either function scope or global scope. Variables declared with var are subject to hoisting, where declarations are moved to the top of their enclosing function or global context. This can lead to unexpected behavior as var variables can be accessed before they are formally declared within the same scope.

let Scope:

The let keyword introduces block-scoped variables. These variables are confined to the nearest enclosing block, whether it's a loop, a conditional statement, or a function. Unlike var, let variables are not hoisted to the top of their block. They are only accessible after their declaration within the block, promoting clearer scoping and preventing scope-related issues.

const Scope:

The const keyword introduces block-scoped variables with the additional characteristic of being constants. This means their value cannot be reassigned after their initial assignment. Similar to let, const variables are block-scoped and are only accessible after their declaration within the block. They ensure controlled scoping and are particularly useful when maintaining constant values throughout their scope.

In modern JavaScript development, using let and const is recommended over var due to their more precise scoping and alignment with contemporary programming practices.


Question-02:Use Cases of null and undefined.

null:

The value null represents the intentional absence of any object value. It is often used when you want to indicate that a variable or object property has no meaningful value or hasn't been initialized yet. For example, you might set a variable to null when you want to clear its previous value.

undefined:

The value undefined typically indicates that a variable or object property has been declared but hasn't been assigned a value. It is often the default value for uninitialized variables. JavaScript assigns undefined to variables that have been declared but haven't been explicitly given a value.

Understanding the use cases of null and undefined is essential for writing clean and robust JavaScript code.


Question-03:What is a REST API?

A REST API, or Representational State Transfer Application Programming Interface, is a set of principles and constraints for designing web services. It allows different software systems to communicate over the internet using standard HTTP methods.

Key features of REST APIs include a stateless architecture, resource-based URLs, and uniform interfaces. They are widely used for building web applications and enabling interactions between various services.
